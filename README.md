# 趣剪（nfun-cv）

本地优先的桌面**剪贴板历史**工具。监听**整个系统剪贴板**（不依赖键盘 hook）：`Ctrl/Cmd+C`、右键复制、其它应用写入均可捕获；支持**文本与图片**，并提供搜索、预览、复用与本地内容分析。

| 项 | 说明 |
| --- | --- |
| 产品名 | 趣剪 |
| 工程名 | `nfun-cv` |
| 技术栈 | Tauri 2 · Rust · Vue 3 · TypeScript |
| 参考产品 | [cv-fun](../cv-fun) |

## 功能

- **全量剪贴板监听**：后台轮询系统剪贴板（非键盘监听），覆盖快捷键与右键复制
- **文本 + 图片**：文本入库；图片保存为 PNG，可预览并复制回剪贴板
- **历史管理**：搜索、选中预览、复制、删除、清空
- **内容分析**：时间戳 / JSON / URL / 英文单词 / 文本统计；图片显示尺寸
- **高性能存储**：SQLite（WAL）存元数据，图片落盘；按内容 hash 去重
- **托盘常驻**：关闭窗口隐藏到托盘，可重新打开

## 存储设计

```text
~/Library/Application Support/NfunCv/   # macOS 示例
├── config.json
├── history.db                 # SQLite + WAL
└── images/
    └── <id>.png
```

为何这样选：

| 方案 | 评价 |
| --- | --- |
| 单 JSON 大文件 | 简单，大图历史时读写差 |
| **SQLite + 图片文件（当前）** | 检索/去重/截断快，图片不塞进 DB |
| 纯 KV | 列表排序与搜索成本更高 |

## 使用

1. 启动趣剪  
2. 任意复制文本或图片  
3. 在列表中搜索、预览、双击/点「复制」写回剪贴板  
4. 关闭窗口 → 托盘仍在；托盘可打开或退出
